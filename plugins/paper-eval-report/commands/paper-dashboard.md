`$ARGUMENTS`에 지정된 폴더 또는 여러 논문의 평가 결과를 읽어,
비교 요약 대시보드 Word 문서를 생성한다.

## 실행 순서

### Step 1. 입력 수집

`$ARGUMENTS`가 폴더 경로인 경우: 해당 폴더의 `*_평가보고서_*.md` 또는 `*_평가보고서_*.docx` 파일 전체를 수집한다.
`$ARGUMENTS`가 없는 경우: 워크스페이스(mnt/Research/)에서 `*_평가보고서_*` 파일을 자동으로 탐색한다.

평가 보고서가 없는 경우: 사용자에게 먼저 `/evaluate-paper`로 논문을 평가할 것을 안내한다.

### Step 2. 데이터 추출

각 평가 보고서에서 아래 항목을 추출한다:

- 논문 제목 / 저자 / 출판연도
- 확인된 연구 설계
- 적용 지침 목록
- 준수율 (%) 및 엄격 준수율 (%)
- 주요 미충족 항목 (상위 3개)
- GRADE 등급 (SR/메타분석만)
- 종합 평가 등급 (AMSTAR 2 신뢰도 또는 동등 지표)

### Step 3. 대시보드 Word 문서 생성

docx 라이브러리로 아래 구성의 Word 문서를 생성한다:

**[표지]**
- "논문 평가 요약 대시보드"
- 생성일, 포함된 논문 수

**[1. 비교 요약 표]**
각 논문 1행: 제목 | 설계 | 준수율 | 엄격 준수율 | GRADE | 주요 문제점

준수율에 따른 행 색상:
- 90% 이상 → 연초록 (`"E2EFDA"`)
- 70–89% → 연노랑 (`"FFF2CC"`)
- 70% 미만 → 연빨강 (`"FCE4D6"`)

**[2. 논문별 미충족 항목 히트맵]**
행: 논문, 열: 주요 지침 항목, 셀: Yes/Partial/No 색상

**[3. 항목별 충족 현황]**
각 주요 항목별로 전체 논문 중 충족 비율 제시

**[4. 공통 개선 권고사항]**
여러 논문에서 반복되는 미충족 항목을 우선순위별로 정리

### Step 4. 저장 및 전달

저장 경로: `[워크스페이스]/paper_dashboard_YYYYMMDD.docx`
`mcp__cowork__present_files`로 파일을 사용자에게 전달한다.
채팅에는 포함된 논문 수, 평균 준수율, 공통 취약점 상위 3개만 요약한다.
